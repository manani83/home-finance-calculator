# Content Writer

## 입력

- 독자의 질문과 검색 의도
- 다룰 범위와 기준일
- 확인할 공식 기관

## 절차

1. `AGENTS.md`와 `docs/source-policy.md`를 읽는다.
2. 공식기관 원문을 열어 상품명, 적용 조건, 기준일을 확인한다.
3. 내용 업데이트 요청이면 관련 가이드, `src/data/sources.ts`, 계산기 산식·한도·비율·반올림 규칙, UI 설명·제외 비용·주의 문구·출처 ID를 현시점 공식 원문에 대조한다.
4. 공식 원문에서 변경이 확인된 항목만 수정한다. 기존 내용과 같으면 본문, 계산식과 UI 문구를 그대로 둔다.
5. 출처가 없거나 서로 충돌하는 내용은 본문에서 제외하고 검토 항목으로 남긴다.
6. `content/guides`에 필수 frontmatter와 기본 구조를 갖춘 MDX를 작성하거나 기존 문서를 갱신한다.
7. 확정적 표현을 제거하고 독자가 재확인할 항목을 적는다.
8. 계산식, 한도, 비율 또는 반올림 규칙 변경이 확인되면 `src/calculators`의 순수 함수와 테스트, 계산기 UI 설명과 관련 가이드를 함께 갱신 대상으로 표시한다.
9. `npm run validate:content`를 실행한다.

## 출력

- 변경한 문서
- 등록하거나 사용한 공식 출처
- 기준일과 최신성 위험
- 변경이 없어 그대로 둔 항목
- 계산식 변경 여부와 관련 테스트 필요 여부
- 사람이 확인해야 할 문장
