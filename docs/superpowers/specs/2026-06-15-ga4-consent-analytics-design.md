# GA4 Consent Analytics Design

## 목표

방문자가 분석 수집에 동의한 경우에만 Google Analytics 4를 로드하고, 계산기 입력값과 계산 결과는 분석 도구로 전송하지 않는다.

## 구성

- `NEXT_PUBLIC_GA_MEASUREMENT_ID`가 설정된 환경에서만 분석 동의 UI를 활성화한다.
- 최초 방문 시 분석 쿠키의 목적을 설명하는 배너와 `동의`, `거부` 버튼을 표시한다.
- 동의 상태는 기능상 필요한 로컬 스토리지 항목 하나에 저장한다.
- 동의한 경우에만 Next.js `GoogleAnalytics` 컴포넌트를 렌더링한다.
- 거부하거나 선택하지 않은 경우 Google 스크립트와 분석 쿠키를 로드하지 않는다.
- 계산기 컴포넌트에는 GA 이벤트 호출을 추가하지 않는다. 페이지 조회 외에 금액, 금리, 계산 결과, 오류 내용 또는 폼 상호작용을 전송하지 않는다.

## 사용자 제어

사용자는 개인정보처리방침에서 분석 수집 목적, 수집 항목, Google의 정보 처리 방식과 거부 방법을 확인할 수 있다. 브라우저의 사이트 데이터에서 동의 저장값과 분석 쿠키를 삭제하면 선택을 초기화할 수 있음을 설명한다.

## 개인정보처리방침

- 시행일을 변경일로 갱신한다.
- Google Analytics 사용 목적과 수집 가능한 방문 정보를 설명한다.
- 계산기 입력값과 결과는 수집하지 않는다고 명시한다.
- 동의 전에는 분석 태그를 로드하지 않는다고 명시한다.
- Google 파트너 사이트 정보 사용 안내와 Analytics 차단 브라우저 부가기능 링크를 제공한다.

## 테스트

- 측정 ID가 없으면 동의 배너와 GA 스크립트가 렌더링되지 않는다.
- 측정 ID가 있고 선택 전이면 배너만 표시된다.
- 동의하면 선택을 저장하고 GA 컴포넌트를 렌더링한다.
- 거부하면 선택을 저장하고 GA 컴포넌트를 렌더링하지 않는다.
- 개인정보처리방침에 분석 목적, 계산기 정보 제외와 거부 방법이 표시된다.
- 계산기 소스에 GA 이벤트 전송 호출이 없음을 검증한다.
