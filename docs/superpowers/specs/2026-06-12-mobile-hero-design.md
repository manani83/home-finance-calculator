# 모바일 메인 히어로 최적화 설계

## 목표

390px 전후의 모바일 화면에서 메인 제목이 화면을 과도하게 차지하지 않도록 줄이고, 핵심 기능을 빠르게 파악할 수 있는 시각적 리듬을 만든다. 데스크톱 히어로의 문구와 배치는 유지한다.

## 선택안

비교 시안의 B안인 핵심어 강조형을 적용한다.

- 모바일 제목: `전세금과 대출 이자를 한눈에 계산하세요`
- `전세금`, `대출 이자`를 각각 별도 요소로 감싸 브랜드 녹색으로 강조한다.
- 모바일 제목 크기는 `36px`, 줄 높이는 `1.1` 수준으로 설정한다.
- 제목 줄바꿈은 세 줄이 되도록 모바일 전용 표시를 사용한다.
- 데스크톱에서는 기존 문구 `전세보증금과 대출 이자를 바로 계산하세요`와 기존 크기를 유지한다.

## 보조 문구

모바일 설명은 `예상 계산값과 공식 확인 자료를 한곳에서 제공합니다.`로 간결하게 표시한다. 데스크톱 설명은 현재 문구를 유지한다.

개인정보 안내 카드의 모바일 문구는 다음처럼 정리한다.

- 라벨: `브라우저에서만 계산`
- 본문: `입력한 금액과 금리는 외부로 전송되지 않습니다.`

데스크톱 카드의 기존 상세 정보와 확인일은 유지한다.

## 구현 방식

- `page.tsx`에 데스크톱용 제목·설명과 모바일용 제목·설명을 각각 둔다.
- CSS 미디어 쿼리로 한쪽만 표시해 화면 크기에 따른 문구를 명확하게 제어한다.
- 모바일 제목의 강조어에는 별도 클래스를 사용한다.
- 모바일 히어로 상단·하단 여백과 버튼 간격을 소폭 줄여 첫 화면에 제목, 설명, 버튼과 개인정보 카드 일부가 균형 있게 보이게 한다.
- 화면 판독기에서 동일한 의미가 중복 낭독되지 않도록 숨겨진 버전은 `display: none` 상태로 유지한다.

## 접근성

- 녹색은 정보나 상태를 구분하는 유일한 수단으로 사용하지 않고 장식적 강조에만 사용한다.
- 강조 단어는 색을 제거해도 전체 문장으로 의미가 전달된다.
- 브랜드 녹색과 히어로 배경의 텍스트 대비를 확인한다.
- 제목을 확대하거나 320px 화면에서 보더라도 잘리거나 가로 스크롤이 생기지 않아야 한다.

## 검증

- 홈 페이지 테스트에서 데스크톱과 모바일 제목·설명 문구가 모두 존재하는지 확인한다.
- CSS 테스트에서 모바일 제목 크기, 강조색과 표시 전환 규칙을 확인한다.
- 390px 및 320px 브라우저 폭에서 제목 줄바꿈, 버튼 배치, 가로 넘침을 확인한다.
- 데스크톱에서 기존 제목과 히어로 배치가 유지되는지 확인한다.
- 전체 테스트, 린트, 타입 검사, 콘텐츠 검사와 프로덕션 빌드를 실행한다.
