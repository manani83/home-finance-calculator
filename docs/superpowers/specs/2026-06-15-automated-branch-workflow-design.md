# Automated Branch Workflow Design

## Goal

Codex가 코드 또는 문서 작업을 완료할 때 브랜치 생성, 커밋, 푸시, PR 생성을 기본 완료 흐름으로 자동 수행한다.

## Workflow

1. 작업 시작 전에 `codex/<작업명>` 형식의 브랜치를 생성한다.
2. 변경을 완료하고 프로젝트의 필수 검증 명령을 모두 통과한 뒤 관련 파일만 커밋한다.
3. 현재 작업 브랜치를 `origin`에 푸시한다.
4. 변경 요약과 검증 결과가 포함된 PR을 생성하고 사람의 검토를 기다린다.

## Safety Boundaries

- 필수 검증이 실패하면 커밋, 푸시 또는 PR 생성으로 진행하지 않는다.
- Git 인증, 원격 저장소 또는 PR 도구 문제로 자동화가 중단되면 성공한 마지막 단계와 실패 원인을 보고한다.
- 사용자 변경이나 작업과 무관한 파일을 커밋하지 않는다.
- `master` 또는 운영 브랜치에 직접 병합하지 않는다.
- PR을 자동 병합하거나 운영 환경에 직접 배포하지 않는다.
- 금융 콘텐츠 또는 계산식 변경 PR에는 기존의 공식 출처, 확인일, 계산식 변경 여부, 테스트 결과와 사람의 확인 항목을 포함한다.

## Verification

`AGENTS.md`의 기존 안전 원칙과 완료 조건을 유지하면서 자동 브랜치 작업 규칙이 추가되었는지 확인한다. Markdown 구조와 Git diff를 검토하고 저장소의 필수 검증 명령을 실행한다.
