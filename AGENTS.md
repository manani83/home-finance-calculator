# AGENTS.md

## 프로젝트 목적

이 프로젝트는 한국 전세, 임대차, 전세대출 관련 계산기와 공식자료 기반 가이드를 제공한다. 금융상품을 중개하거나 특정 상품 가입을 유도하지 않는다.

## 필수 안전 원칙

1. 금융·법률 설명은 공식 원문을 우선하고 출처 ID와 확인일을 남긴다.
2. 출처를 확인할 수 없는 수치나 조건을 추측하지 않는다.
3. 계산 결과를 `예상 계산값`으로 표시하고 실제 심사 결과와 구분한다.
4. 계산기 입력값을 API, 로그, 쿠키, 로컬 스토리지 또는 분석 도구로 보내거나 저장하지 않는다.
5. 주민등록번호, 계좌번호, 신용점수, 소득자료와 계약서 등 민감정보를 수집하는 폼을 만들지 않는다.
6. 금융 콘텐츠와 계산식 변경은 PR과 사람의 검토를 거친다. Codex가 운영 배포를 직접 수행하지 않는다.

## 공식 출처 우선순위

1. 국가법령정보센터, 금융위원회, 금융감독원
2. HF, HUG, SGI와 은행연합회
3. 개별 금융기관의 공식 상품설명서와 공지

언론, 블로그와 커뮤니티는 공식 원문을 찾기 위한 단서로만 사용한다.

## 계산기 규칙

- 산식은 `src/calculators`의 순수 함수로 작성한다.
- UI 컴포넌트에 금융 계산식을 직접 작성하지 않는다.
- 새 동작은 실패 테스트를 먼저 작성하고 정상값, 0, 음수, 경계값과 소수 금리를 검증한다.
- 금액은 원 단위로 받고 표시 단위를 명시한다.
- 반올림 규칙과 계산에 포함되지 않는 비용을 화면에 설명한다.

## 콘텐츠 규칙

- 가이드는 `content/guides/*.mdx`에 저장한다.
- `title`, `description`, `slug`, `updatedAt`, `reviewStatus`, `sources`, `disclaimer`를 포함한다.
- `reviewStatus: reviewed` 문서만 공개한다.
- 출처는 `src/data/sources.ts`에 먼저 등록한다.
- 한 줄 요약, 대상 독자, 핵심 설명, 체크리스트, 공식 출처와 면책을 제공한다.
- `무조건 가능`, `반드시 승인`, `누구나 가능`, `100% 보장`, `최저금리 확정` 등 확정적 표현을 쓰지 않는다.

## 완료 조건

작업 완료 전에 다음 명령을 모두 실행한다.

```bash
npm test
npm run lint
npm run typecheck
npm run validate:content
npm run build
```

## 브랜치와 PR 자동화

Codex는 코드 또는 문서 변경 작업을 완료할 때 다음 순서를 기본 흐름으로 자동 수행한다.

1. 작업 시작 전에 `codex/<작업명>` 형식의 브랜치를 생성한다.
2. 변경을 완료하고 위 필수 검증을 모두 통과한 뒤 작업 관련 파일만 커밋한다.
3. 현재 작업 브랜치를 `origin`에 푸시한다.
4. 변경 요약과 검증 결과를 포함한 PR을 생성하고 사람의 검토를 기다린다.

다음 조건에서는 이후 Git 단계를 진행하지 않고 성공한 마지막 단계와 중단 원인을 보고한다.

- 필수 검증 명령이 하나라도 실패한 경우
- Git 인증 또는 `origin` 원격 저장소를 사용할 수 없는 경우
- PR 생성 도구나 GitHub 연동을 사용할 수 없는 경우

사용자가 만든 변경이나 현재 작업과 무관한 파일은 커밋하지 않는다. `master` 또는 운영 브랜치에 직접 병합하지 않으며, PR 자동 병합과 운영 배포를 수행하지 않는다.

금융 콘텐츠를 변경한 PR에는 변경 페이지, 공식 출처, 확인일, 계산식 변경 여부, 테스트 결과와 사람이 확인할 부분을 적는다.
